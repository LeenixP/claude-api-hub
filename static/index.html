Getting Started

Set up API Hub in 4 simple steps and start routing your Claude requests.

1
Install & Start
Install API Hub globally and launch the server
Terminal 
npm install -g claude-api-hub

# Start the server
claude-api-hub
The server starts on port 3456 by default. Use --port to change it.
2
Configure Claude Code
Point Claude Code to your API Hub gateway
~/.claude/settings.json 
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:3456"
  }
}
Replace localhost:3456 with your actual server address if running remotely.
3
Add a Provider
Register your API providers in the Dashboard
Example: Adding DeepSeek 
# Via API (or use the Dashboard "Add" button)
curl -X POST http://localhost:3456/api/config/providers \
  -H "Content-Type: application/json" \
  -d '{
    "name": "DeepSeek",
    "baseUrl": "https://api.deepseek.com/v1",
    "apiKey": "sk-your-key",
    "models": ["deepseek-chat", "deepseek-reasoner"],
    "defaultModel": "deepseek-chat",
    "enabled": true
  }'
Click + Add in the Providers section for a visual form. Use Fetch from API to auto-detect models.
4
Set Up Aliases
Map Claude tiers (haiku/sonnet/opus) to any provider model
Alias Mapping Example 
# In the Dashboard "Alias Mapping" section:
#   Haiku  → deepseek-chat       (fast, cheap)
#   Sonnet → gpt-4o              (balanced)
#   Opus   → claude-opus-4-5-20250414  (powerful)

# Or via API:
curl -X PUT http://localhost:3456/api/aliases \
  -H "Content-Type: application/json" \
  -d '{
    "haiku": "deepseek-chat",
    "sonnet": "gpt-4o",
    "opus": "claude-opus-4-5-20250414"
  }'
When Claude Code requests claude-3-5-haiku, API Hub routes it to your mapped model.

Frequently Asked Questions

API Hub supports both OpenAI-compatible and Anthropic-native formats. Toggle between them per provider using the protocol badge on each provider card.
When a request fails (timeout, rate limit, server error), API Hub automatically retries with the next available provider that supports the requested model. Configure multiple providers with overlapping models for best resilience.
Yes. API Hub supports key pooling — add multiple keys separated by commas in the API Key field. Requests are distributed across keys using round-robin to avoid rate limits.
In the Alias Mapping section, each tier (Haiku/Sonnet/Opus) has a timeout field in seconds. Set longer timeouts for Opus (complex tasks) and shorter ones for Haiku (quick lookups). The default is 300 seconds.
Optional. Set ADMIN_PASSWORD environment variable to enable password protection for the dashboard. API requests use the x-api-key header for authentication.
Configuration is stored in providers.json in the working directory. Request logs are kept in memory (last 200 entries) and optionally written to disk when File Log is enabled.
