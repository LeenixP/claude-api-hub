# 项目治理

本文档描述 claude-api-hub 项目的治理结构、决策流程和贡献者晋升路径。

## 项目所有权

- **创建者**：LeenixP
- **许可证**：MIT License
- **代码托管**：[GitHub](https://github.com/LeenixP/claude-api-hub)

## 决策流程

项目决策分为三个层级：

### 日常决策

- 代码审查、小功能合并、文档修正
- 由任意维护者独立决定
- 无需额外审批

### 技术决策

- 架构调整、依赖升级、API 变更
- 需在 Issue 或 PR 中公开讨论
- 至少一名维护者审核通过后方可执行

### 重大变更

- 破坏性变更、许可证变更、项目方向调整
- 必须发起公开讨论，收集社区反馈
- 由维护者团队达成共识后决定

## 角色定义

### 维护者（Maintainer）

维护者负责项目的日常运营和技术方向：

- 审核并合并 Pull Request
- 发布新版本
- 管理 Issue（分类、标记、关闭）
- 维护文档和发布说明
- 指导新贡献者

当前维护者：LeenixP

### 贡献者（Contributor）

贡献者通过以下方式参与项目：

- 提交 Pull Request
- 报告 Issue（Bug 或功能请求）
- 改进文档
- 参与代码审查和讨论

任何遵循本项目 [行为准则](./CODE_OF_CONDUCT.md) 和 [贡献指南](./CONTRIBUTING.md) 的人都可以成为贡献者。

## 贡献者晋升路径

```
首次贡献（提交 PR 或报告 Issue）
        ↓
持续贡献（累计 5+ 合并的 PR）
        ↓
受邀成为协作者（Collaborator）——获得代码审查权限
        ↓
长期贡献且表现突出 → 由现有维护者提名为维护者
```

### 各阶段说明

| 阶段 | 条件 | 权限 |
|------|------|------|
| 贡献者 | 提交 PR 或 Issue | 参与讨论、提交代码 |
| 协作者 | 5+ 合并的 PR，质量稳定 | 审查 PR、管理 Issue、推送至非保护分支 |
| 维护者 | 长期贡献，技术判断力强，社区认可 | 合并代码、发布版本、管理项目设置 |

## 争议解决

1. **Issue 讨论**：所有技术争议首先在相关 Issue 或 PR 中公开讨论
2. **维护者裁决**：若讨论 7 天后仍未达成一致，由维护者做出最终决定
3. **社区投票**：涉及项目方向、许可证等重大争议，可发起社区投票，维护者拥有最终决定权

## 行为准则

本项目遵循 [Contributor Covenant](https://www.contributor-covenant.org/) 行为准则。

所有参与者（包括维护者和贡献者）都应：

- 使用友善和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

违反行为准则的行为可向 leenixp@gmail.com 报告。

## 安全响应

安全漏洞报告将在 **48 小时内** 收到回复。详细流程请参阅 [SECURITY.md](./SECURITY.md)。
